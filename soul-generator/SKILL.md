# Soul Generator

収集した情報から OpenClaw 用 Soul.md を生成する。

## コンセプト

「この人に相談できるサービス」を作るために、人物の：
- 専門知識とナレッジ
- 思考スタイルと意思決定の軸
- コミュニケーションの特徴
- 相談への乗り方

を構造化し、AI が再現できる形式（Soul.md）にまとめる。

## ワークフロー

```
Phase 1: 素材受け取り（情報源の収集・整理）
Phase 2: 情報抽出（素材から人物像を抽出）
Phase 3: 構造化（テンプレートに沿って整理）
Phase 4: Soul.md 生成（レビュー用ドラフト出力）
Phase 5: ユーザー確認・調整
```

## Phase 1: 素材受け取り

ユーザーから以下のいずれか（複数可）を受け取る：

| 素材タイプ | 処理方法 |
|-----------|---------|
| テキストメモ・インタビュー記録 | そのまま分析 |
| URL（ブログ、SNS、記事） | WebFetch で取得して分析 |
| ファイルパス（.md, .txt, .pdf） | Read で読み込んで分析 |
| 口頭での説明 | ヒアリングで深掘り |

### 素材が不十分な場合のヒアリング

最低限以下が分からないと Soul は作れない。不足分はユーザーに質問する：

1. **この人は誰か**: 名前、肩書き、経歴の要点
2. **何の専門家か**: 専門領域とその深さ
3. **どう相談に乗るか**: 相談スタイル、得意な相談テーマ
4. **どんな話し方をするか**: トーン、口癖、説明の仕方

## Phase 2: 情報抽出

素材を読み込み、以下の観点で情報を抽出する：

### 抽出する情報カテゴリ

- **Identity**: 経歴、実績、立ち位置
- **Expertise**: 専門知識、独自の方法論、ノウハウ
- **Thinking Style**: 意思決定の軸、問題解決の癖
- **Communication Style**: 話し方、教え方、接し方
- **Consulting Approach**: 得意テーマ、進め方、NGテーマ
- **Values**: 信念、譲れないポイント

### URL がある場合

WebFetch で URL の内容を取得し、以下を抽出：
- プロフィール情報
- 発言内容から推測される思考スタイル
- 専門テーマと知識の深さ
- 文体・トーンの特徴

## Phase 3: 構造化

`references/soul-template.md` のテンプレートに沿って情報を整理する。

### 品質基準

- **具体性**: 「マーケティングに詳しい」ではなく「BtoB SaaS のリードジェネレーション、特にコンテンツマーケティング戦略に強い」
- **再現性**: AI がこの Soul を読んだだけで、その人"っぽい"回答ができるレベル
- **口癖・フレーズ**: 最低3つは具体的なフレーズを含める
- **境界の明確さ**: 「やること」だけでなく「やらないこと」も定義する

## Phase 4: Soul.md 生成

テンプレートに基づいて Soul.md を生成し、出力する。

### 出力先

```
{プロジェクトディレクトリ}/souls/{人物名-kebab-case}.md
```

例: `souls/haruki.md`

### 出力前チェック

生成した Soul.md を以下の観点で自己チェック：

1. [ ] テンプレートの全セクションが埋まっているか
2. [ ] 「具体的なフレーズ・口癖」が3つ以上あるか
3. [ ] 専門領域が曖昧でなく、具体的に書かれているか
4. [ ] 「やらないこと・NGテーマ」が明記されているか
5. [ ] この Soul を読んだ AI が、その人"らしく"振る舞えるか

## Phase 5: ユーザー確認

生成した Soul.md をユーザーに提示し、フィードバックを受ける。

確認ポイント：
- 「この人っぽさ」が再現できているか
- 足りない情報・修正したい箇所はないか
- 追加したい得意テーマやNGテーマはないか

フィードバックを反映して最終版を出力する。
